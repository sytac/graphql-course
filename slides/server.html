Data mapping


              query {
                  posts {
                    title
                    body
                  }
              }
          

              schema {
                query: Query
              }
            
              type Query {
                posts: [Post]!
              }

Data mapping


              const resolvers = {
                Query: {
                  posts: () => fetch('/v1/posts')
                }
              };
          

              type Query {
                posts: [Post]!
              }

              type Post {
                title: String!
                body: String!
              }

Data mapping


                const resolvers = {
                  Post: {
                    title: post => post.title, // Default implementation
                    body: post => post.body
                  }
                };
            

                type Post {
                  title: String!
                  body: String!
                }

Data mapping


                const resolvers = {
                  ...
                  Post: {
                    author: post =>
                      fetch('/v1/author/' + post.authorId)
                  },
                };
            

                type Post {
                  title: String!
                  author: Author
                }

                type Author {
                  name: String!
                }

Data mapping


                  query {

Data mapping


                const resolvers = {
                  Query: {
                    posts: (_, { limit }) => 
                      fetch('/v1/posts').then(posts => posts.slice(0, limit))
                  }
                };
              

                  type Query {
                    posts(limit: Int): [Post]!
                  }

Authentication?

Cookies or header tokens

Caching?

  • Cache data at the resolver level, or cache the underlying REST calls
  • Persisted queries

Query abuse?

  • Max depth
  • Query complexity cost limits
  • Persisted queries

Profiling and Monitoring?

  • Apollo engine (proprietary)
  • Apollo tracing

Deprecation?


              type ExampleType {
                newField: String
                oldField: String @deprecated(reason: "Use `newField`.")
              }

Versioning?

  • Prefer an append-only scheme
  • Version the endpoints (E.g. /v1/graphql)

TL;DL

  • GraphQL servers resolve queries using types, fields and resolvers
  • GraphQL based servers have patterns for common concerns of API's in production
