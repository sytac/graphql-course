What is GraphQL

Describe your data


					type Project {
						name: String
						tagline: String
						contributors: [User]
					}

Ask for what you want


						{
							project(name: "GraphQL") {
								tagline
							}
						}

Get predictable results


					{
						"project": {
							"tagline": "A query language for APIs"
						}
					}

Why should you choose GraphQL?

Faster frontend development

  • Spend less time composing data from different endpoints and focus on what should be displayed
  • Superb Out-of-the-box tooling and libraries … makes interacting with the backend a lot smoother
  • Same protocol for queries, mutations and real-time updates

Less burden on API teams

  • No need to make specific endpoints as each client can request specifically what they need
  • Since all data is available, there is less focus on specific use cases
  • No need to explicitly document APIs – the schema is the documentation

  • GraphQL is a query (and schema definition) language
